# 第一步：AI 数据标采交付机器人 MVP 启动清单

## 目标

先做一个最小可跑通版本，不追求全自动，也不要求接入公司平台 API。

第一步的目标是确认：

- 机器人第一批服务哪些场景。
- 哪些数据放飞书。
- 哪些内容进入知识库。
- 哪些信息必须脱敏或不能进入机器人。
- 哪些输出可以直接帮助运营减少重复工作。

## 第一阶段只做 5 个高频场景

建议第一版 MVP 只覆盖以下场景：

1. 新需求结构化
2. 平台操作引导
3. 项目进度摘要
4. 质量返修说明生成
5. 结算前复核清单

先不要做：

- 自动报价
- 自动推荐供应商
- 自动读取历史单价
- 跨客户项目查询
- 全量质量趋势分析

这些后面都可以做，但第一版会引入权限、数据准确性和敏感信息风险。

## 你需要先收集的材料

### 1. 真实但可脱敏的客户需求样例

准备 10 条左右，不要带客户名、项目名、真实价格。

格式示例：

```text
客户原话：
我们想做一批图片标注，主要是道路场景，想训练识别车辆和行人。

运营理解：
自动驾驶图像 2D 框标注需求，需要确认数据量、类别体系、验收标准、样例数据和交付格式。
```

### 2. 平台操作节点

列出客户在平台必须完成的动作。

建议先整理成：

```text

适用角色：
触发条件：
用户需要准备什么：
平台上要点哪里：
常见错误：
完成后下一步：
```

优先整理这些：

- 需求提交
- 数据上传
- 安全信息填报
- 样例数据提交
- 方案确认
- 进度查看
- 验收确认

### 3. 运营后台动作

列出运营在平台要做的动作。

建议先整理：

```text

```

优先整理这些：

- 需求评估
- 工具配置
- 供应商选配
- 竞价确认
- 批次创建
- 质量问题登记
- 返修跟进
- 结算复核

### 4. 质量返修样例

准备 5 条脱敏样例。

格式示例：

```text
飞书沟通原文摘要：
这批数据里车辆框偏大，部分遮挡车辆没有标。

标准化质量事件：
缺陷类型：框偏移、漏标
严重级别：中
影响范围：批次 BATCH-001 抽检样本
关联规则：车辆 2D 框标注规则
返修要求：收紧框边界，补充遮挡车辆标注
```

### 5. 结算前检查项

整理你们运营结算前最常看的事项。

建议包括：

- 项目是否验收通过
- 批次是否全部完成
- 返修是否关闭
- 质量事件是否关闭
- 交付量是否一致
- 供应商结算口径是否确认
- 是否存在扣减项
