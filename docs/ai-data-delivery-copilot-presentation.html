产品定位
Problems

三类核心问题

客户侧
  • 不知道需求文档怎么写。
  • 不清楚上传、安全填报、样例提交流程。
  • 想了解单价、效率、质量，但涉及权限。
  • 项目进度反馈零散。
运营侧
  • 需求在飞书产生，最终回平台建单。
  • 质量、返修在飞书沟通，结算回平台闭环。
  • 平台数据和沟通内容缺少统一知识层。
  • 经验依赖个人。
管理侧
  • 延期风险难以及时发现。
  • 供应商效率与质量表现分散。
  • 返修、规则更新、结算争议难复盘。
  • 成熟能力难产品化输出。

端到端流程
Capabilities

核心能力模块

1. 需求顾问

口语需求 → 标准需求文档，追问缺失字段。

2. 平台操作向导

引导上传、安全填报、任务提交、验收结算。

3. 方案与规则助手

生成规则草案、质量检查点和验收口径。

4. 供应商选配

基于质量、效率、产能和价格风险辅助决策。

5. 项目进度助手

查询项目阶段、批次状态、延期风险。

6. 批次与效率查询

预估效率、实际效率、逾期和人天投入。

7. 质量协同监督

质量事件、规则条款、返修说明和复盘建议。

8. 结算复核

交付量、返修关闭、质量扣罚、结算口径。

9. FAQ 问答

流程、操作、财务、规则、供应商高频问题。

MVP Validation

已实现的 MVP 能力

需求结构化

  • 从口语描述提取需求类型、数据模态、场景、数据量。
  • 主动追问需求方、验收标准、优先级。
  • 信息齐全后生成结构化需求文档。

规则与规范问答

  • 按关键词匹配规则表。
  • 输出规则名称、版本、适用场景和质量度量。
  • 支持规则版本化管理。

项目与批次查询

  • 列出进行中项目。
  • 根据需求ID关联项目状态。
  • 按团队/供应商查看任务和效率。

质量事件分析

  • 按项目、规则、事件类型、严重级别筛选。
  • 列出 P0/P1 事件和处理状态。
  • 严格基于表中数据回答。

Security

权限与数据安全

内容分级

等级内容
L0 公共FAQ、通用规则、平台操作说明
L1 内部通用运营 SOP、工具配置说明、质检方法
L2 项目受限需求、项目、任务批次、质量事件
L3 敏感受限客户名称、供应商报价、绩效、结算明细
L4 禁止进入 AI原始数据、未脱敏样本、个人信息、密钥

执行原则

知识库按子团队拆分，Agent 实例隔离部署。

系统提示词注入权限边界声明。

敏感数据不进入公共知识库。

对外演示和作品集使用虚拟数据。

权限模型
Access Matrix

角色权限矩阵

角色可查看不可查看
外部客户自己组织的需求、项目、L0 规则其他客户、供应商报价、内部 SOP
内部运营负责或协作的项目全链路数据非负责项目的敏感受限数据
供应商分配给自己的批次、规则、质检反馈其他供应商、竞价、成本
质检负责人授权项目的质量事件、规则、返修非授权项目质量数据
管理层汇总数据和授权明细非必要敏感受限明细
关键原则机器人不能拿全量数据后自行判断是否能说，应只接收已经按权限过滤后的结果。

度量体系
Risks

风险与应对

数据泄露风险

由平台或飞书控制权限，敏感数据不进公共知识库，对外演示使用虚拟数据。

AI 编造风险

所有数字和状态必须来自表记录，无匹配使用标准空结果话术，查询问题优先结构化数据。

流程落地风险

每个输出绑定下一步动作，运营确认后写入正式系统，分阶段从半自动过渡到自动化。

产品泛化风险

抽象为需求、项目、批次、质量、返修、结算等通用对象，对外版支持字段、流程、权限可配置。
